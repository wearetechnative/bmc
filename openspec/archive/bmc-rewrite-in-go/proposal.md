## Why

BMC is currently a collection of bash scripts with multiple runtime dependencies (gum, jq, awk, jsonify-aws-dotfiles, assumego, aws-mfa). This makes installation fragile, cross-platform behaviour inconsistent (macOS vs Linux date handling, shell differences), and the codebase hard to test and extend. A full rewrite in Go produces a single self-contained binary with zero runtime dependencies for core functionality.

## What Changes

- **BREAKING**: Config file moves from `~/.config/bmc/config.env` (sourced bash) to `~/.config/bmc/config.toml` (TOML)
- **BREAKING**: `profsel` now requires a shell wrapper function to set `AWS_PROFILE` in the parent shell; install via `bmc install-shell-integration`
- All commands rewritten in Go using AWS SDK Go v2 — no aws CLI needed for core operations
- `gum` TUI replaced by bubbletea + bubbles (charmbracelet Go libraries)
- `aws-mfa` (broamski) MFA refresh logic inlined — bmc now manages STS session tokens directly
- `assumego` (Granted) replaced by native STS AssumeRole + AWS federation URL construction
- `jsonify-aws-dotfiles`, `jq`, `awk` eliminated — AWS config parsed natively in Go
- New command: `bmc doctor` — system check for all prerequisites and configuration
- New command: `bmc install-shell-integration` — installs profsel shell wrapper
- Shell completion generated by cobra (replaces `gencompletions` command)
- Distribution via GoReleaser: GitHub releases, Homebrew tap, Nix flake

## Capabilities

### New Capabilities

- `aws-profile-selection`: Interactive fuzzy AWS profile selection grouped by account group, with MFA session refresh
- `aws-console`: Open AWS console in browser via STS AssumeRole + federation URL (replaces assumego)
- `ec2-operations`: List, find, stop/start EC2 instances and toggle InstanceScheduler tag
- `ec2-connect`: Connect to EC2 instances via SSH (using existing .ssh/config ProxyCommand) or SSM session
- `ecs-connect`: Interactive ECS cluster/service/task/container selection and execute-command shell
- `mfa-session`: STS GetSessionToken with MFA, writes credentials to ~/.aws/credentials (compatible with aws-mfa format)
- `config-management`: Read/write ~/.config/bmc/config.toml; migrate from legacy config.env
- `prerequisites-check`: Lazy per-command prerequisite validation with actionable Nix/apt/brew install instructions; `bmc doctor` for full system check
- `shell-integration`: Install profsel shell wrapper into .zshrc/.bashrc via `bmc install-shell-integration`
- `distribution`: Single binary distribution via GoReleaser (GitHub releases, Homebrew tap, Nix flake buildGoModule)

### Modified Capabilities

## Impact

- **Removes runtime deps**: gum, jq, awk, jsonify-aws-dotfiles, assumego, aws-mfa
- **Retains optional deps**: ssh (for ec2connect SSH), aws CLI v2 + session-manager-plugin (for ec2connect SSM and ecsconnect)
- **Breaking config change**: users must migrate `~/.config/bmc/config.env` → `~/.config/bmc/config.toml`
- **Breaking shell change**: `source bmc profsel` pattern replaced by shell wrapper + `eval`
- **~/.aws/credentials**: write format preserved — compatible with all other AWS tooling
- **Nix flake**: updated to use `buildGoModule` instead of installing bash scripts
- **release.sh**: version bump and changelog process unchanged, binary name unchanged (`bmc`)
